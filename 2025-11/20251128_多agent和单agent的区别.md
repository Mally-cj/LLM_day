### **单智能体 和 多智能体的核心区别？**

单智能体 = 一个大脑做所有决策，工具只是执行；流程必然串行。工程上表现为：一个角色 prompt + 一个 memory + 单一推理循环。

多智能体 = 多个大脑分工协作，每个角色独立思考，可真正并行。工程上表现为：多个角色 prompt + 各自独立的 memory + 显式的消息通信结构。

⚠️ “多个大模型 API ≠ 多智能体”。
判断标准是：是否存在多个独立思考单元，而不是模型数量。

---

**举例：写一篇综述论文**

**单智能体（Single-Agent）**

像是 *1 个人从头干到尾*，所有推理集中在一个大脑里，因此步骤只能顺序执行：

1. 想题目
2. 拆任务
3. 搜文献
4. 读论文
5. 写初稿
6. 自己审自己

→ **优点：逻辑统一；缺点：容易过载、不能并行。**

**多智能体（Multi-Agent）**

像是 *一个团队*，每个角色都有自己的“小脑子”（独立思考能力），彼此通过消息协作：

- Planner：拆任务
- 多个 Researcher：**并行**查不同方向的文献
- Analyzer：对比证据
- Writer：写初稿
- Reviewer：审稿查错

→ **优点：可并行、高扩展性；缺点：需要协调。**
